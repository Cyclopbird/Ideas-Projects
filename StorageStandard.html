Storage and Structuring Standard for Ideas & Projects

This document outlines the recommended system for organizing, storing, and linking scientific ideas and projects to ensure easy discoverability, clarity, and interoperability.

1. Unique IDs

Each idea/project must have a unique ID to serve as a reference point. Example format:

Idea_[Topic]_[Number]

Example: Idea_Fusion_001

2. File Naming & Location

  • Use .html files named exactly after the ID. E.g. Idea_Fusion_001.html
  • Store all idea files in a dedicated folder, e.g. /Ideas-Projects/
  • Include a central index.html listing all IDs with links and short descriptions.

3. Content Structure

  • Title: Clear and descriptive
  • Author: Name and affiliation
  • Description: Short summary
  • Detailed content: Structured with headings
  • Tags: Keywords or topics for categorization

4. Metadata

Use machine-readable metadata (e.g. JSON-LD with schema.org) embedded in the page header or body to facilitate automated indexing by search engines and AI.

5. Versioning and Updates

  • Keep version history within the document or in an associated log
  • Update the central index accordingly

6. Cross-Linking and Clustering

Link related ideas by referencing their unique IDs within the content. Use tags or categories to group ideas thematically.

7. Hosting and Accessibility

Host the files on platforms supporting static sites (e.g. GitHub Pages) to ensure global availability.

8. International and Open Standards

Adhere to international open data and semantic web standards to maximize interoperability.
